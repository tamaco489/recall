# MD Linter — Markdown 静的解析・自動修正スキル

`markdownlint-cli2` を使って Markdown ファイルの警告を検出・修正する。

## 処理ステップ概要

| Step | 内容                  | 概要                                                              |
| ---- | --------------------- | ----------------------------------------------------------------- |
| 1    | 対象パスの確定        | ユーザー指定パス、またはプロジェクトルート配下全体を対象にする    |
| 2    | 初回スキャン          | `markdownlint-cli2` で現在の警告一覧を取得する                    |
| 3    | 自動修正              | `--fix` フラグで自動修正可能な警告を一括修正する                  |
| 4    | Prettier フォーマット | `prettier --write` でテーブル整列などのフォーマットを適用する     |
| 5    | 再スキャン            | 修正後の残存警告を確認する                                        |
| 6    | 手動修正              | 自動修正できなかった警告をファイルを読んで1件ずつ修正する         |
| 7    | 最終確認              | 再度スキャンして警告がゼロになったか確認する                      |
| 8    | 結果報告              | 修正件数・残存警告をユーザーに報告する                            |

## 対象パスの決定ルール

- ユーザーが明示的にパスを指定した場合: そのパスを使用する
- 指定がない場合: プロジェクトルート配下の全 Markdown ファイルを対象にする (`.claude/` を含む)

## 設定ファイル

プロジェクトルートの `.markdownlint.json` を設定ファイルとして使用する。
ファイルが存在しない場合は `markdownlint-cli2` のデフォルト設定が適用される。
ルールを追加・変更する場合はこのファイルを編集する。

## 処理フロー

### Step 1: 対象パスの確定

ユーザーのリクエストから対象パスを決定する。

```text
パス指定あり → そのパスを使用 (例: docs/)
パス指定なし → **/*.md を対象にする
```

### Step 2: 初回スキャン

対象パスに対して markdownlint-cli2 を実行し、警告の全量を把握する。

```bash
markdownlint-cli2 "対象パス/**/*.md"
```

警告が0件の場合はそのまま Step 7 へ進んで完了を報告する。

### Step 3: 自動修正

`--fix` フラグで自動修正可能な警告を一括修正する。

```bash
markdownlint-cli2 --fix "対象パス/**/*.md"
```

自動修正の対象となる主なルールは `.claude/rules/markdown/auto-fix-rules.md` を参照。

### Step 4: Prettier フォーマット

`markdownlint --fix` の後、Prettier でテーブル整列・インデント統一などのフォーマットを適用する。

```bash
npx prettier --write "対象パス/**/*.md"
```

- プロジェクトルートに `.prettierrc` があればその設定を使用する
- なければ Prettier のデフォルト設定 (テーブルパイプ整列など) が適用される
- `proseWrap: preserve` 相当の挙動となり、本文の折り返しは変更されない

### Step 5: 再スキャン

自動修正後に再度スキャンして残存警告を確認する。

```bash
markdownlint-cli2 "対象パス/**/*.md"
```

残存警告が0件の場合は Step 7 へ進む。

### Step 6: 手動修正

自動修正できなかった警告を1件ずつ手動で修正する。

#### 修正の進め方

1. 残存警告をファイルパスごとにグループ化する
2. ファイルを Read ツールで読み込む
3. 該当行を確認し、ルールに従って Edit ツールで修正する
4. 全件修正後に Step 5 を再実行して確認する

#### 手動修正が必要な主なルール

手動修正が必要なルールは `.claude/rules/markdown/manual-fix-rules.md` を参照。

#### 対象ファイルが多い場合

警告が多いときは、ファイルの修正優先度を以下の順に処理する:

1. 警告件数が多いファイルから順に修正する
2. 同一ファイル内の警告は上から順に修正する (行番号が大きい修正を先にすると行ズレが起きるため)

### Step 7: 最終確認スキャン

全修正後に最終スキャンを実行し、警告件数が0であることを確認する。

```bash
markdownlint-cli2 "対象パス/**/*.md"
```

警告が残っている場合は Step 6 に戻って修正を続ける。

### Step 8: 結果報告

完了後、ユーザーに以下を報告する:

- 修正前の警告総数
- 自動修正した件数 (--fix による修正)
- 手動修正した件数
- 修正後の残存警告数 (0 件であることが理想)
- 修正したファイル一覧
- 解消できなかった警告がある場合はその内容と理由

## 注意事項

- `--fix` は上書き保存するため、git の変更差分として確認できる
- `.markdownlint.json` で無効化されているルールは警告対象外
- 修正時は既存のコンテンツを壊さないよう慎重に行う
