# CLAUDE.md

Claude Code のセッションメモリと作業ルールを一元管理するプロジェクトです。

## 制約事項

> [!IMPORTANT]
>
> - **即時性は求めない。時間をかけてでも根拠に基づく正確なアウトプットを行う**
> - 公式ドキュメントや関連資料の調査はメインコンテキストを汚さないよう、別途調査用エージェントに委譲する

## 構成

- `.claude/rules/github/` — コミット・PR のルールとテンプレート
- `.claude/skills/` — 各スキルの定義ファイル

## 開発コマンド

| コマンド            | 内容                                            |
| ------------------- | ----------------------------------------------- |
| `make setup`        | 依存パッケージインストール & モデルダウンロード |
| `make install`      | 依存パッケージのみインストール                  |
| `make download-model` | 埋め込みモデルのみダウンロード                |
| `make build`        | TypeScript をビルドする                         |
| `make dev`          | ウォッチモードでビルドする                      |
| `make start`        | ビルドして MCP サーバーを起動する               |
| `make clean`        | ビルド成果物 (dist/) を削除する                 |
| `make up`           | Qdrant コンテナを起動する                       |
| `make down`         | Qdrant コンテナを停止・削除する                 |
| `make logs`         | Qdrant コンテナのログを表示する                 |
| `make rebuild`      | Qdrant コンテナをキャッシュなしで再ビルド       |
| `make health`       | Qdrant のヘルスチェックを行う                   |
| `make collections`  | Qdrant のコレクション一覧を表示する             |
| `make help`         | 全コマンドのヘルプを表示する                    |

## 実行可能なスキル

| スキル名       | 用途                                                    | トリガー例                                          |
| -------------- | ------------------------------------------------------- | --------------------------------------------------- |
| `md-linter`    | Markdown ファイルを静的解析し、警告を自動・手動修正する | 「md を lint して」「markdownlint を直して」        |
| `smart-commit` | 差分をグループ化してコミット・push・PR 作成まで行う     | 「コミットして」「変更を push して」「PR 作成して」 |
